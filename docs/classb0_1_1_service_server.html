Detailed Description

template<typename TReq, typename TRep, bool notifyGraph = true>
class b0::ServiceServer< TReq, TRep, notifyGraph >

Service server.

This class wraps a ZeroMQ REP socket. It will use Node::bindSocket() to automatically announce the socket name to resolver.

If using a callback, when a request is received, it will be handed to the callback, as long as a spin method is called (e.g. Node::spin(), Node::spinOnce() or ServiceServer::spinOnce()).

You can directly read requests and write replies to the underlying socket, by using ServiceServer::poll(), ServiceServer::read() and ServiceServer::write().
